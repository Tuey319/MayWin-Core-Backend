# Information Backup
**ISO/IEC 27001:2022 — Control 8.13**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Protect against data loss through regular, tested backups of MayWin information assets.

## Backup Strategy

### PostgreSQL Database (AWS RDS)
| Parameter | Value |
|---|---|
| Backup type | Automated daily snapshots |
| Retention period | 7 days |
| Point-in-time recovery | Enabled (5-minute RPO) |
| Storage encryption | AWS-managed KMS |
| Cross-region copy | Recommended for production |

**Manual backup before migrations**: Always take a manual RDS snapshot before running `npm run migration:run` in production.

### S3 Artifacts
| Parameter | Value |
|---|---|
| Versioning | Enabled |
| Lifecycle policy | Delete after 90 days |
| Storage class | S3 Standard |
| Encryption | AES-256 (SSE-S3) |

### Application Code
| Parameter | Value |
|---|---|
| Repository | GitHub — full history maintained |
| Branch strategy | `main` (production), feature branches |
| Recovery | Clone and deploy — no data loss (stateless) |

### Audit Logs
| Parameter | Value |
|---|---|
| Storage | Database table + CSV export capability |
| Backup | Included in RDS automated backup |
| Retention | Minimum 12 months |

## Backup Testing
- **Monthly**: Verify RDS automated backup is completing successfully (check AWS console).
- **Quarterly**: Perform a test restore of the latest RDS snapshot to a test instance. Verify data integrity.
- **Annually**: Full recovery test — restore from backup, run migrations, verify application starts.

## Recovery Procedure
See `5.30-ICT-READINESS-FOR-BUSINESS-CONTINUITY.md` and `BACKUP.md`.

- `restore-db.sh`: Database restore script.
