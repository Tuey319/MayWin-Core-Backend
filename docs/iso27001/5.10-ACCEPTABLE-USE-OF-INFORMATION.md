# Acceptable Use of Information and Other Associated Assets
**ISO/IEC 27001:2022 — Control 5.10**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Define rules for the acceptable use of information assets within the MayWin platform to protect data integrity and confidentiality.

## Information Assets in Scope
| Asset | Classification | Acceptable Use |
|---|---|---|
| Nurse personal data (name, email, LINE ID) | Confidential | Scheduling operations only — no export to third parties |
| Schedule data | Internal | View/edit by authorized unit staff only |
| Audit logs | Restricted | Read by `ORG_ADMIN` only — no deletion permitted |
| JWT tokens | Secret | Single-user session only — must not be shared |
| API keys (Gemini, LINE) | Secret | Server-side use only — never exposed to client |
| Database credentials | Secret | Backend service only — never logged or exposed |
| OTP codes | Confidential | Single-use, 10-minute TTL — must not be shared |

## Prohibited Uses
- Exporting nurse personal data outside the platform without explicit authorization.
- Sharing JWT access tokens between users or sessions.
- Using production data in development or test environments.
- Logging credentials, OTPs, or API keys in any log output.
- Using the LINE Bot integration for purposes other than nurse scheduling communication.

## Enforcement
Violations are recorded in the audit log (`AuditLogsService`) and reviewed by the Technical Lead. Repeated violations escalate to the project owner.
