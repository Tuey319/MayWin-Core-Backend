# Addressing Information Security in Supplier Agreements
**ISO/IEC 27001:2022 — Control 5.20**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Ensure supplier agreements include appropriate information security requirements to protect MayWin data.

## Requirements by Supplier Type

### Cloud Infrastructure (AWS)
- AWS Shared Responsibility Model applies: AWS secures the infrastructure; MayWin secures the application, data, and access management.
- RDS encryption at rest enabled. SSL connections required (`PGSSLMODE=require`).
- S3 bucket is private — no public access allowed.
- IAM principle of least privilege: application role has only required permissions (RDS connect, S3 read/write to specific bucket, SFN start execution).

### AI Services (Google Gemini)
- API keys stored server-side only — never exposed to clients.
- No nurse PII (full name, employee ID) included in Gemini prompts unless operationally required.
- If personal data is ever sent, ensure Google's data processing addendum is accepted.

### Messaging (LINE Corporation)
- LINE Messaging API terms of service accepted.
- Webhook signature verification enforced on all incoming requests.
- LINE user IDs treated as Confidential (5.12).

### Email (SMTP)
- SMTP connection must use TLS (`secure: true` or `STARTTLS`).
- OTP email content must not include additional personal data beyond the user's first name.

## Review
Supplier agreements and security posture reviewed annually or when a supplier reports a significant security incident.
