# Contact with Special Interest Groups
**ISO/IEC 27001:2022 — Control 5.6**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Maintain membership in or contact with security forums and special interest groups to stay current on threats, vulnerabilities, and best practices.

## Relevant Groups and Sources

| Group / Source | Relevance | Frequency |
|---|---|---|
| NVD (National Vulnerability Database) | CVEs affecting Node.js, NestJS, TypeORM, PostgreSQL | Monitor monthly via `npm audit` |
| GitHub Security Advisories | Alerts for direct dependencies in `package.json` | Automated via Dependabot / GitHub alerts |
| OWASP | Top 10 web application risks — input validation, injection, auth | Review annually |
| AWS Security Bulletins | Vulnerabilities in AWS services used (RDS, S3, Lambda, SFN) | Subscribe to AWS Health Dashboard |
| LINE Developers Blog | SDK updates and security notices for the webhook integration | Monitor quarterly |
| Node.js Security Releases | Runtime security patches | Monitor at nodejs.org/en/blog/vulnerability |

## Actions
- Run `npm audit` before every production deployment.
- Subscribe to GitHub security alerts on the repository.
- Review OWASP Top 10 annually and update `THREAT_MODEL.md` accordingly.
- Maintain dependency update cadence — patch within 30 days of critical CVE notification.
