# Responsibilities After Termination or Change of Employment
**ISO/IEC 27001:2022 — Control 6.5**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Ensure information security responsibilities remain in effect after a user's employment ends or their role changes.

## Continuing Obligations
Even after termination or departure from the project, all former users remain bound by:
- Confidentiality obligations regarding nurse personal data they had access to.
- Non-disclosure of system credentials, API keys, or architectural details.
- Applicable data protection legislation (Thailand PDPA).

## Immediate Actions on Termination

### For Nurses / General Users
- [ ] Deactivate account: set `is_active = false` in the users table.
- [ ] Revoke LINE link token if one exists (mark `used_at = NOW()`).
- [ ] Remove unit membership record if applicable.
- [ ] Audit log entry created for account deactivation.

### For ORG_ADMIN / UNIT_MANAGER
All steps above, plus:
- [ ] Review audit logs for the past 30 days for any unusual activity before departure.
- [ ] Transfer any active pending schedules to another authorized manager.

### For Developers
All steps above, plus:
- [ ] Revoke GitHub repository access within 24 hours.
- [ ] Rotate `JWT_SECRET` if they had access to `.env.production`.
- [ ] Rotate all other secrets they may have had access to.
- [ ] Remove from AWS IAM if applicable.

## Verification
The `ORG_ADMIN` confirms all off-boarding steps are complete and records completion in the audit log with action `OFFBOARD_USER`.
