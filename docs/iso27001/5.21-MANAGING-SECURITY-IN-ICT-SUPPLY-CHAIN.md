# Managing Information Security in the ICT Supply Chain
**ISO/IEC 27001:2022 — Control 5.21**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Manage security risks arising from the software supply chain, including open-source dependencies and third-party libraries.

## Dependency Security

### Direct Dependencies of Risk Note
| Package | Version | Risk | Mitigation |
|---|---|---|---|
| `bcrypt` | ^6.0.0 | Password hashing — critical | Pin version; monitor for CVEs |
| `passport-jwt` | ^4.0.1 | Auth flow | Monitor JWT library advisories |
| `typeorm` | (transitive) | ORM / SQL safety | Use parameterised queries only — no raw SQL |
| `@line/bot-sdk` | ^10.6.0 | External messaging | Monitor LINE SDK releases |
| `@google/generative-ai` | ^0.24.1 | AI integration | Monitor Google AI SDK releases |
| `class-validator` | ^0.14.3 | Input validation | Keep up to date; validation is security-critical |
| `helmet` | latest | Security headers | Keep up to date |

### Controls
- `npm audit` run before every production deployment.
- `package-lock.json` committed to repository — ensures reproducible builds.
- Dependabot or equivalent configured on GitHub repository to alert on known CVEs.
- No dependencies installed with `--ignore-scripts` disabled — review `postinstall` scripts for new packages.
- No use of packages with zero weekly downloads or abandoned maintenance.

## Process for Adding New Dependencies
1. Check npm advisory database and GitHub for known vulnerabilities.
2. Verify the package is actively maintained (last publish < 12 months).
3. Review the package's own dependencies for transitive risks.
4. Document in a PR review comment if the package handles security-sensitive operations.
