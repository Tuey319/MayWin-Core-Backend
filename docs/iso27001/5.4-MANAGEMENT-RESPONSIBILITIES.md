# Management Responsibilities for Information Security
**ISO/IEC 27001:2022 — Control 5.4**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Define management obligations to support the implementation and operation of information security controls within the MayWin backend system.

## Scope
Applies to all technical leads, project owners, and administrators of the MayWin platform.

## Responsibilities

### Technical Lead / Project Owner
- Ensure all developers follow secure coding standards documented in this repository.
- Review and approve changes to authentication, authorization, and database configuration.
- Ensure security findings from audits are tracked and resolved within agreed timescales.
- Ensure only authorized personnel have access to production environment variables and AWS credentials.

### Developers
- Follow the secure development guidelines in `docs/BACKEND_DEVELOPER_GUIDE.md`.
- Report suspected security incidents immediately via the incident response process (`INCIDENT_RESPONSE_PLAN.md`).
- Never commit secrets, credentials, or environment files to version control.

### All Personnel
- Complete security awareness training before being granted system access.
- Acknowledge and comply with the `ACCEPTABLE_USE_POLICY.md`.

## Accountability
The Technical Lead is accountable for ensuring this policy is enforced and reviewed at least annually or following any significant change to the system architecture.
