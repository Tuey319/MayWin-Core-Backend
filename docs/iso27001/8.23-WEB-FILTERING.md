# Web Filtering
**ISO/IEC 27001:2022 — Control 8.23**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Manage access to external websites and web resources to reduce exposure to malicious content and data leakage.

## Developer Workstation Web Filtering
- Developers should use corporate or secure DNS (e.g. Cloudflare 1.1.1.1 or Google 8.8.8.8) to benefit from malicious domain blocking.
- Browser extensions that intercept HTTPS traffic should not be installed on machines used for MayWin development.
- npm registry access restricted to `https://registry.npmjs.org` — no private registries without Technical Lead approval.

## API Outbound Connections
The MayWin backend only makes outbound HTTP connections to an approved allowlist:

| Destination | Protocol | Purpose |
|---|---|---|
| `*.amazonaws.com` | HTTPS | AWS services (RDS, S3, SFN, Lambda) |
| `api.line.me` | HTTPS | LINE Messaging API |
| `generativelanguage.googleapis.com` | HTTPS | Gemini AI |
| SMTP host (configurable) | SMTP/TLS | OTP email delivery |

**No other outbound HTTP connections are made by the application.** Any new external integration must be reviewed and added to this allowlist.

## Enforcement Recommendation
In production (Lambda/ECS), configure a VPC with:
- NAT Gateway for controlled outbound internet access.
- Security Group egress rules permitting only the above destinations.
- VPC Endpoints for AWS service communication (no internet traversal needed for AWS APIs).
