# Disciplinary Process
**ISO/IEC 27001:2022 — Control 6.4**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Establish a formal process for handling violations of information security policies by MayWin system users.

## Violation Categories

### Minor Violations
- Accessing another user's schedule data without authorization (but no exfiltration).
- Using the system for minor non-scheduling purposes.
- Failing to log out from shared devices.

**Response**: Verbal warning from Unit Manager. Audit log reviewed. Refresher training on Acceptable Use Policy.

### Moderate Violations
- Sharing login credentials with another person.
- Attempting to access data outside one's authorized scope.
- Downloading or exporting nurse data without authorization.

**Response**: Written warning. Account temporarily suspended pending review. Audit log exported and reviewed by ORG_ADMIN. Technical Lead notified.

### Serious Violations
- Deliberate unauthorized access to personal data.
- Sharing system credentials externally.
- Tampering with audit logs.
- Using the system to harass or discriminate against staff.

**Response**: Immediate account deactivation (`is_active = false`). Escalation to hospital administration. Legal action if applicable. Incident documented in `INCIDENT_RESPONSE_PLAN.md`.

## Investigation Process
1. Audit log exported by `ORG_ADMIN` for the relevant time period.
2. Technical Lead reviews logs to establish facts.
3. User notified and given opportunity to respond.
4. Decision made and recorded.
5. If credentials were compromised, rotate affected secrets immediately (independent of disciplinary outcome).
