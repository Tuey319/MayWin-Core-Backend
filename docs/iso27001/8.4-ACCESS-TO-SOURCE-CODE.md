# Access to Source Code
**ISO/IEC 27001:2022 — Control 8.4**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Manage and restrict access to the MayWin application source code to prevent unauthorized modification.

## Repository Access Control

| Role | Access Level | Permissions |
|---|---|---|
| Technical Lead | Admin | Push to main, merge PRs, manage branch protection |
| Developer | Write | Push to feature branches, create PRs |
| Academic Reviewer | Read | Read-only access to repository |
| External Contributor | None by default | Must be explicitly added |

## Branch Protection Rules (GitHub)
- `main` branch is protected:
  - Direct push disabled — all changes via Pull Request.
  - At minimum 1 approving review required.
  - Status checks (build/lint) must pass before merge.
- No force-push to `main` or `production` branches.

## Secret Protection
- `.env`, `.env.*`, and any credential files are in `.gitignore`.
- GitHub secret scanning enabled — alerts on accidentally committed secrets.
- `JWT_SECRET`, database credentials, and API keys are stored in GitHub Secrets for CI/CD — not in repository files.

## Audit Trail
- All commits are attributed to authenticated GitHub accounts.
- PR review history is maintained permanently.
- Force-push is disabled — history cannot be rewritten on protected branches.

## Access Review
Repository access list reviewed quarterly. Developers who leave the project are removed within 24 hours (see `5.11-RETURN-OF-ASSETS.md`).
