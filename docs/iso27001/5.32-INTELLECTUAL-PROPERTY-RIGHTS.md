# Intellectual Property Rights
**ISO/IEC 27001:2022 — Control 5.32**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Ensure compliance with intellectual property rights and the legal use of software, libraries, and data.

## Open Source Licenses in Use

All dependencies listed in `package.json` are used in accordance with their licenses:

| License Type | Key Packages | Obligation |
|---|---|---|
| MIT | NestJS, TypeORM, bcrypt, helmet, passport, class-validator, rxjs | Retain copyright notice |
| Apache 2.0 | AWS SDK packages (`@aws-sdk/*`) | Retain copyright and license notice |
| ISC | Various utility packages | Retain copyright notice |
| BSD | Reflect-metadata | Retain copyright notice |

## Obligations
- License files for all dependencies are maintained in `node_modules/` — do not strip.
- No GPL-licensed packages used — avoids copyleft obligations on proprietary code.
- The MayWin application source code is proprietary and must not be redistributed without authorization.

## Proprietary Assets
- Solver logic (`src/core/solver/`) and scheduling algorithms are proprietary.
- Nurse data and schedule data belong to the operating healthcare organization and must not be used for any purpose other than scheduling.
- LINE Bot conversation data belongs to the respective nurse users.

## AI-Generated Content
- Code assistance (where used) does not transfer copyright obligations — the generated code is reviewed and owned by the development team.
- Gemini API outputs are used operationally (message parsing) — no copyright claims apply to AI-interpreted scheduling data.

## Review
License compliance reviewed when adding new dependencies or redistributing the software.
