# Monitoring and Review of Supplier Services
**ISO/IEC 27001:2022 — Control 5.22**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Regularly monitor and review supplier services to ensure continued security and performance commitments are met.

## Review Schedule

| Supplier | Review Activity | Frequency |
|---|---|---|
| AWS | Review IAM permissions, CloudWatch alarms, Security Hub findings | Monthly |
| AWS RDS | Check backup status, SSL enforcement, patch level | Monthly |
| Google Gemini API | Review API usage, check for SDK updates | Quarterly |
| LINE Messaging API | Review webhook logs, check for credential rotation need | Quarterly |
| npm dependencies | `npm audit` + Dependabot alerts | Every deployment |
| GitHub | Review repository access list, branch protection rules | Quarterly |

## Monitoring Indicators
- **AWS**: No unexpected IAM policy changes; no unusual S3 access patterns; RDS CPU/memory within bounds.
- **LINE Bot**: Webhook rejection rate (invalid signatures) should be zero under normal operation. Any spike indicates spoofing attempts → investigate and rotate credentials.
- **Gemini API**: Key exhaustion rate — all keys should have available quota. Rapid exhaustion may indicate a compromised key.
- **Auth**: Monitor login failure rate via request logs. Spike may indicate credential stuffing.

## Actions on Service Failure
If a supplier experiences a security incident affecting MayWin:
1. Assess data exposure — refer to `INCIDENT_RESPONSE_PLAN.md`.
2. Rotate relevant credentials immediately.
3. Notify affected users if their data was exposed.
4. Document in incident register.
