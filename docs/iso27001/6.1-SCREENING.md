# Screening
**ISO/IEC 27001:2022 — Control 6.1**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Verify the background and suitability of individuals before granting access to the MayWin system and its data.

## Screening Requirements by Role

| Role | Access Level | Screening Required |
|---|---|---|
| `ORG_ADMIN` | Full system access, audit logs, staff management | Identity verification + hospital employment confirmation |
| `UNIT_MANAGER` | Schedule creation, staff management within unit | Identity verification + unit assignment confirmation |
| `NURSE` | Own schedule and preferences only | LINE account verification (LINE link token) |
| Developer | Source code, may access staging | Identity verification + signed NDA |

## Screening Process
1. **Identity verification**: Confirm full name matches employment records.
2. **Role confirmation**: Verify the requested access level is appropriate for the individual's job function.
3. **Acknowledgement**: Individual must acknowledge the `ACCEPTABLE_USE_POLICY.md` and `src/ACCEPTABLE_USE_POLICY.md` before account activation.
4. **Account creation**: Staff accounts created via `POST /api/v1/core/staff` by an `ORG_ADMIN` or `UNIT_MANAGER` — system generates a temporary password and sends welcome email.

## Technical Enforcement
- `is_active = false` by default until activation confirmed.
- Temporary passwords generated using `crypto.randomBytes` (cryptographically secure).
- First login requires password change (policy — not yet enforced in code; to be implemented).

## Record Keeping
Account creation is recorded in the audit log (`CREATE_STAFF` action) with the actor's ID and name.
