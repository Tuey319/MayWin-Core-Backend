# Threat Intelligence
**ISO/IEC 27001:2022 — Control 5.7**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Collect, analyse, and act on threat intelligence relevant to the MayWin platform to proactively reduce risk.

## Threat Landscape

### Platform-Specific Threats
| Threat | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Credential stuffing on `/auth/login` | High | High | Rate limiting (`@Throttle` 10/15min), 2FA OTP |
| JWT token theft | Medium | High | Short-lived tokens (8h), HTTPS only via Helmet HSTS |
| LINE webhook spoofing | Medium | High | HMAC-SHA256 signature verification (`timingSafeEqual`) |
| SQL injection via ORM | Low | Critical | TypeORM parameterised queries — no raw SQL |
| Exposed secrets in source control | Was High | Critical | `.env` gitignored, secrets rotated |
| Brute force OTP | Medium | High | Throttle + `timingSafeEqual` |
| Unauthorized schedule manipulation | Medium | High | `RolesGuard` + `@Roles()` on all write endpoints |
| API enumeration / IDOR | Medium | Medium | Organization-scoped queries in all services |

### External Intelligence Sources
- **OWASP Top 10 2021**: A01 Broken Access Control, A02 Cryptographic Failures, A07 Auth Failures — all directly addressed.
- **MITRE ATT&CK**: Credential Access (T1110 Brute Force) → rate limiting; Exfiltration (T1041) → audit logging.
- **CVE feeds**: Monitored via `npm audit` and GitHub Dependabot.

## Review Cadence
- Monthly: `npm audit` + review new CVEs for dependencies.
- Quarterly: Review threat table above and update mitigations.
- After incident: Update threat model with lessons learned.
