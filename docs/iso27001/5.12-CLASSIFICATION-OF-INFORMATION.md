# Classification of Information
**ISO/IEC 27001:2022 — Control 5.12**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Classification Scheme

| Level | Label | Description | Examples |
|---|---|---|---|
| 1 | **Public** | No harm if disclosed | API documentation, README, health endpoint |
| 2 | **Internal** | For project team only | Schedule data, unit configuration, solver logic |
| 3 | **Confidential** | Restricted to authorized roles | Nurse personal data, LINE IDs, audit logs |
| 4 | **Secret** | Strictly controlled | Passwords, JWT secret, API keys, DB credentials |

## Data Classification Map

| Data Type | Classification | Storage | Access Control |
|---|---|---|---|
| Nurse name, email | Confidential | PostgreSQL (encrypted at rest via AWS RDS) | `NURSE` (own), `UNIT_MANAGER`, `ORG_ADMIN` |
| LINE user ID | Confidential | PostgreSQL | `UNIT_MANAGER`, `ORG_ADMIN` |
| Schedule assignments | Internal | PostgreSQL | Unit members |
| Audit logs | Confidential | CSV file + database | `ORG_ADMIN` only |
| OTP codes | Secret | PostgreSQL (short-lived) | System only — never exposed in API response |
| JWT tokens | Secret | Client-side (memory/storage) | Issuing user only |
| Database credentials | Secret | Environment variables | Backend process only |
| Gemini API keys | Secret | Environment variables | Backend process only |
| LINE Bot secrets | Secret | Environment variables | Backend process only |

## Handling Rules
- **Confidential** data must never appear in error messages returned to the client.
- **Secret** data must never be logged, even at debug level.
- Data at rest (PostgreSQL on AWS RDS) is encrypted using AWS-managed keys.
- Data in transit must use TLS (enforced by Helmet HSTS header and AWS load balancer configuration).
