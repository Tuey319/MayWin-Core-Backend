# Information Security for Use of Cloud Services
**ISO/IEC 27001:2022 — Control 5.23**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Establish controls for the acquisition, use, and management of cloud services to maintain security of MayWin data.

## Cloud Services in Use

| Service | Provider | Region | Purpose |
|---|---|---|---|
| RDS PostgreSQL | AWS | ap-southeast-1 | Primary database |
| S3 | AWS | ap-southeast-1 | Schedule artifacts storage |
| Lambda | AWS | ap-southeast-1 | Solver execution |
| Step Functions | AWS | ap-southeast-1 | Scheduling workflow orchestration |
| Gemini AI | Google Cloud | Global | Nurse message interpretation |

## Security Controls

### Identity and Access
- AWS IAM roles used — no long-lived access keys where avoidable.
- Application uses environment-variable-injected credentials (`AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`).
- Principle of least privilege: each IAM role has only the permissions required for its function.

### Data Protection
- RDS encryption at rest: AWS-managed KMS keys.
- S3 bucket: `Block Public Access` enabled; server-side encryption (AES-256) enabled.
- All AWS API calls use HTTPS (SDK default).
- `PGSSLMODE=require` enforced for all database connections.

### Availability
- RDS: Multi-AZ deployment recommended for production.
- S3: 99.999999999% durability (11 nines) — used as artifact store.
- Lambda: Stateless — auto-scales; no single-point-of-failure.
