# User Endpoint Devices
**ISO/IEC 27001:2022 — Control 8.1**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Protect information processed on, stored on, or accessed via user endpoint devices.

## Devices in Scope
| Device Type | Users | Data Accessed |
|---|---|---|
| Developer laptops | Developers / Technical Lead | Source code, staging environment |
| Admin workstations | ORG_ADMIN, UNIT_MANAGER | MayWin web application (schedule data, staff data) |
| Nurse smartphones | Nurses | LINE messenger only (no direct system access) |

## Security Requirements

### Developer Laptops
- Full-disk encryption mandatory (BitLocker on Windows, FileVault on macOS).
- OS and browser kept up to date with security patches.
- Antivirus / endpoint protection installed.
- No production `.env` files stored locally — use AWS Secrets Manager.
- Screen lock on idle (≤5 minutes).
- Secure Wi-Fi only — no public hotspots without VPN.

### Admin Workstations
- HTTPS-only access to the MayWin application.
- Browser must not save passwords for admin accounts — use a password manager.
- Log out after each session.
- Do not install browser extensions that could intercept network traffic.

### Nurse Smartphones (LINE Access)
- Nurses should enable LINE's built-in PIN or biometric lock.
- Nurses should not share their LINE account.
- If a phone is lost: notify Unit Manager → revoke LINE link token → re-register if needed.
