# Collection of Evidence
**ISO/IEC 27001:2022 — Control 5.28**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Ensure that evidence related to security incidents is collected, preserved, and maintained in a manner suitable for investigation and legal proceedings.

## Evidence Sources in MayWin

| Source | Location | Retention | Access |
|---|---|---|---|
| Audit logs | `AuditLogsService` CSV + database | Minimum 12 months | `ORG_ADMIN` only |
| HTTP request logs | `RequestLoggerMiddleware` (stdout → CloudWatch) | 90 days (CloudWatch default) | AWS console |
| Authentication events | Captured in request logs (login, OTP verify) | 90 days | AWS console |
| Database change history | TypeORM migration files in `src/database/migrations/` | Permanent (Git) | Repository access |
| AWS CloudTrail | AWS account activity (IAM, S3, RDS, Lambda calls) | 90 days (default) | AWS console |
| LINE Bot event logs | `WebhookController` logger output → CloudWatch | 90 days | AWS console |

## Evidence Preservation Principles
1. **Integrity**: Audit logs are append-only — no `DELETE` or `UPDATE` operations on `audit_logs`.
2. **Chain of custody**: Logs include `actorId`, `actorName`, `action`, `targetId`, and `timestamp`.
3. **Tamper evidence**: AWS CloudTrail provides immutable record of infrastructure access.
4. **Non-repudiation**: JWT authentication ensures each audit log entry is tied to an authenticated user identity (`sub` claim).

## In the Event of an Incident
1. Immediately export relevant audit logs via `GET /api/v1/core/audit-logs?export=csv` (admin only).
2. Preserve AWS CloudWatch log streams for the incident time window — export to S3 before retention expires.
3. Do not modify any logs, database records, or configuration until the investigation is complete.
4. Escalate to law enforcement if criminal activity is suspected — provide preserved logs.
