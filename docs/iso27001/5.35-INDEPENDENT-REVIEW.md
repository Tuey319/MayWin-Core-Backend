# Independent Review of Information Security
**ISO/IEC 27001:2022 — Control 5.35**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Ensure the MayWin information security approach is reviewed independently to verify it remains effective and compliant.

## Review Approach

### Internal Review (Quarterly)
- Technical Lead reviews all security controls against this document set.
- Verify code guards, rate limiting, and logging are functioning in production.
- Run `npm audit` and review findings.
- Review access control matrix (`ACCESS_CONTROL_MATRIX.md`) for accuracy.

### Peer Code Review (Per Pull Request)
- All changes to auth, guards, database queries, and webhook handlers require a second developer review.
- Security checklist in PR template:
  - [ ] No hardcoded credentials
  - [ ] All new endpoints have `@UseGuards(JwtAuthGuard, RolesGuard)`
  - [ ] Input validated via DTO with `class-validator`
  - [ ] No sensitive data in logs

### Academic / External Review
- This codebase is submitted for academic evaluation against ISO/IEC 27001:2022.
- External evaluators have read-only access to this repository.

### Annual Review
- Full review of all policy documents in `docs/iso27001/`.
- Verify supplier certifications (AWS, Google) remain valid.
- Review and update `THREAT_MODEL.md` based on new threats.
- Test business continuity procedures (`5.30`).

## Findings and Actions
All findings from reviews are tracked as GitHub issues with the `security` label and assigned to the Technical Lead.
