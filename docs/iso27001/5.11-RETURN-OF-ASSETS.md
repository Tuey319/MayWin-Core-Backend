# Return of Assets
**ISO/IEC 27001:2022 — Control 5.11**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Ensure all information assets are returned or securely revoked when a team member leaves the project or changes role.

## Off-boarding Checklist

When a developer, administrator, or contractor leaves the project:

### Access Revocation (must be completed within 24 hours)
- [ ] Revoke GitHub repository access
- [ ] Deactivate AWS IAM user and rotate any shared credentials they had access to
- [ ] Rotate `JWT_SECRET` if the departing member had access to `.env.production`
- [ ] Regenerate LINE Bot `channelAccessToken` and `channelSecret`
- [ ] Rotate Gemini API keys if accessible
- [ ] Deactivate the user's account in the MayWin system (`is_active = false` in users table)
- [ ] Remove unit membership records if applicable

### Data Return
- [ ] Confirm no system data (database exports, environment files, API keys) was retained on personal devices.
- [ ] Remove any local `.env` copies from developer machines.

### Documentation
- Record the off-boarding in the access review log (`src/ACCESS_REVIEW_POLICY.md`).
- Audit log entry created for account deactivation.

## Role Changes
If a user's role changes (e.g. `NURSE` → `UNIT_MANAGER`), update their `unit_memberships.role_code` and `user_roles` records. Old access rights must be explicitly revoked — they are not automatically removed by role upgrade.
