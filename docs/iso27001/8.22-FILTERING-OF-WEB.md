# Filtering of Web
**ISO/IEC 27001:2022 — Control 8.22**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Protect the MayWin API from malicious web traffic and prevent access to harmful external resources.

## Inbound Traffic Filtering

### Applied at Application Layer (NestJS)
| Control | Implementation | File |
|---|---|---|
| Security headers | `helmet()` — CSP, X-Frame-Options, X-Content-Type-Options, HSTS, etc. | `src/main.ts` |
| CORS | `enableCors()` — restricted to `ALLOWED_ORIGINS` env var | `src/main.ts` |
| Input validation | `ValidationPipe` with `whitelist: true` — unknown fields stripped | `src/main.ts` |
| Rate limiting | `ThrottlerModule` — 60 req/min/IP global; 10/15min on auth | `src/app.module.ts`, `auth.controller.ts` |
| Request authentication | `JwtAuthGuard` on all non-public endpoints | All controllers |
| Role-based access | `RolesGuard` + `@Roles()` decorators | All controllers |
| Webhook signature | HMAC-SHA256 verification on LINE webhook | `webhook.controller.ts` |

### Recommended at Infrastructure Layer (AWS WAF)
The following should be configured in AWS WAF for production:
- Block requests with SQL injection patterns.
- Block requests with XSS patterns.
- Block IPs with high request rates (complement to ThrottlerModule).
- Geo-blocking: restrict to Thailand (or operational regions) if appropriate.
- AWS Managed Rules: `AWSManagedRulesCommonRuleSet` and `AWSManagedRulesBotControlRuleSet`.

## Outbound Filtering
The application makes outbound requests to:
- AWS services (RDS, S3, Lambda, Step Functions) — internal VPC traffic.
- `api.line.me` — LINE Messaging API.
- `generativelanguage.googleapis.com` — Gemini AI API.
- SMTP server — OTP email delivery.

All outbound connections use HTTPS (TLS) enforced by the SDK/library defaults.

## Content Security Policy
`helmet()` sets default CSP headers. The API serves JSON only — no HTML content rendering that would require a permissive CSP.
