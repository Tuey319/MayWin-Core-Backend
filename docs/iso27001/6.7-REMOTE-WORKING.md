# Remote Working
**ISO/IEC 27001:2022 — Control 6.7**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Protect information accessed and processed when personnel work remotely from the primary work location.

## Scope
Applies to all developers, administrators, and users who access the MayWin system from remote locations (home, hospital wards, mobile devices).

## Remote Access Controls

### For Developers
- Source code accessed only via GitHub over HTTPS — no direct production database access from developer machines.
- Production environment variables (`.env.production`) must never be stored on personal devices. Use AWS Secrets Manager or Parameter Store.
- Local development uses `.env` with non-production credentials only.
- VPN recommended when working from public networks.
- Developer machines must have full-disk encryption enabled.
- `npm install` and development work done in isolated project directories — no system-wide installs of project dependencies.

### For Admins (ORG_ADMIN / UNIT_MANAGER)
- Access the MayWin web application only via HTTPS.
- Do not use the system on shared or public computers.
- Log out after each session — JWT tokens have an 8-hour expiry but manual logout is best practice.
- Enable device lock (screen timeout) on any device used to access the system.

### For Nurses (Remote LINE Access)
- Nurses interact via LINE messenger — LINE's own security (2FA on LINE account) applies.
- Nurses should not share their LINE account with others.
- LINE link tokens expire after 48 hours and are single-use.

## Incident Reporting
If a remote device is lost, stolen, or compromised:
1. Report immediately to the Technical Lead.
2. Technical Lead rotates any credentials potentially accessible from the device.
3. Active JWT sessions cannot be revoked individually — `JWT_SECRET` rotation invalidates all sessions if device compromise is suspected.
4. Log the incident in `INCIDENT_RESPONSE_PLAN.md`.
