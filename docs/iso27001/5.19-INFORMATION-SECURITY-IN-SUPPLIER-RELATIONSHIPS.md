# Information Security in Supplier Relationships
**ISO/IEC 27001:2022 — Control 5.19**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Manage information security risks arising from the use of third-party suppliers and cloud service providers.

## Supplier Register

| Supplier | Service | Data Shared | Security Assurance |
|---|---|---|---|
| Amazon Web Services (AWS) | Cloud hosting, RDS, S3, Lambda, Step Functions | All platform data (encrypted at rest) | ISO 27001, SOC 2 Type II certified |
| Google (Gemini AI API) | AI-powered nurse message interpretation | Anonymised message text | Google Cloud privacy policy, GDPR compliant |
| LINE Corporation | Messaging integration (LINE Bot) | Nurse LINE user IDs, message content | LINE security policies, encrypted in transit |
| Nodemailer / SMTP provider | OTP email delivery | Nurse email addresses | TLS enforced for all SMTP connections |
| GitHub | Source code hosting, CI/CD | Source code (no production data) | SOC 2 certified, 2FA enforced |
| npm / Node.js ecosystem | Runtime dependencies | None | Monitored via `npm audit` and Dependabot |

## Security Requirements for Suppliers
Before integrating a new third-party service:
1. Confirm HTTPS / TLS for all data transfer.
2. Review the supplier's data processing agreement and privacy policy.
3. Determine if nurse personal data (PII) will be transmitted — if yes, ensure PDPA compliance.
4. Document in this register and update `ASSET_INVENTORY.md`.

## Monitoring
- AWS: Monitor via AWS Security Hub and CloudWatch.
- Dependencies: `npm audit` run on each deployment.
- LINE / Gemini: Monitor platform status pages quarterly.
