# Information Deletion
**ISO/IEC 27001:2022 — Control 8.10**
**MayWin Nurse Scheduling Platform**
Last reviewed: 2026-03-30

---

## Purpose
Ensure information is deleted securely when no longer required, in compliance with data protection obligations.

## Retention Schedule

| Data Type | Retention Period | Deletion Method |
|---|---|---|
| Nurse personal data (name, email, LINE ID) | Duration of employment + 1 year | Soft delete (`is_active = false`) then hard delete after retention period |
| Schedule assignments | 3 years (operational history) | Database DELETE via admin tooling |
| Audit logs | Minimum 12 months | Overwrite oldest entries after retention period |
| OTP codes | Used immediately; expire after 10 minutes | Marked `used_at = NOW()` — auto-eligible for purge after 24 hours |
| LINE link tokens | 48-hour expiry + used tokens | Mark `used_at = NOW()` — purge after 7 days |
| Chatbot conversation history | 90 days | Scheduled purge (to be implemented) |
| S3 artifacts (schedule exports) | 90 days | S3 lifecycle policy — auto-delete after 90 days |
| CloudWatch logs | 90 days | CloudWatch log group retention policy |

## Deletion Procedures

### User Account Deletion
1. Set `is_active = false` (immediate — user cannot log in).
2. After retention period: remove from `users`, `unit_memberships`, `user_roles` tables.
3. Anonymise or remove `full_name`, `email` from `workers` table attributes.
4. Log deletion in audit log.

### Data Subject Requests (PDPA / Right to Erasure)
If a nurse requests deletion of their personal data:
1. Verify identity via `ORG_ADMIN`.
2. Assess whether operational records must be retained for legal/regulatory reasons.
3. Delete or anonymise as appropriate.
4. Respond to the data subject within 30 days.

## Technical Implementation
OTP and link token purging can be implemented as a scheduled TypeORM query. Example:
```sql
DELETE FROM maywin_db.auth_otps WHERE used_at IS NOT NULL AND created_at < NOW() - INTERVAL '24 hours';
DELETE FROM maywin_db.line_link_tokens WHERE used_at IS NOT NULL AND created_at < NOW() - INTERVAL '7 days';
```
