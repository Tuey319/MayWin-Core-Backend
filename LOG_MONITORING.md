# Monitoring of Activities

**Control**: ISO/IEC 27002 A.8.16

This document describes the plan for monitoring security-relevant logs generated by the MayWin Core Backend.

## 1. Logged Security Events
The `SecurityLoggerInterceptor` generates logs for every API request, capturing:
- **Authentication**: Login success and failure (`POST /auth/login`).
- **Access Control**: Unauthorized access attempts (401/403 errors).
- **Privileged Actions**: All requests to endpoints, including those guarded by `RolesGuard`.
- **Data Modification**: All `POST`, `PATCH`, `PUT`, `DELETE` requests.

Each log entry includes: `Timestamp`, `Actor` (User ID or Anonymous), `Roles`, `IP Address`, `Action` (HTTP Method and URL), `Result` (Success/Failure), and `Duration`.

## 2. Monitoring and Review
- **Frequency**: Security logs must be aggregated into a centralized logging system (e.g., ELK Stack, CloudWatch Logs) and reviewed at least weekly.
- **Automated Alerts**: Automated alerts should be configured in the logging system to trigger on the following indicators of suspicious activity.

## 3. Indicators of Suspicious Activity (Alerting Thresholds)
- **High rate of failed logins**: >10 failed login attempts from a single IP address in 5 minutes.
- **High rate of 403 Forbidden errors**: >20 authorization failures for a single user in 10 minutes.
- **Anomalous privileged activity**: An `ORG_ADMIN` action occurring outside of normal business hours.
- **Backup/Restore Activity**: Any execution of `restore-db.sh` outside of a planned maintenance window.
- **Log Service Failure**: An alert should trigger if no logs are received from the application for more than 1 hour.
