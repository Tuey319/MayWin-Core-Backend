# Information Asset Inventory

**Control**: ISO/IEC 27002 A.5.9

This document lists the key information and technology assets associated with the MayWin Core Backend.

| Asset ID | Asset Name | Description | Owner/Custodian | Classification |
| :--- | :--- | :--- | :--- | :--- |
| **AS-01** | **Source Code** | The NestJS backend application source code. | Engineering Team | Confidential |
| **AS-02** | **PostgreSQL Database** | The primary data store for all scheduling, user, and configuration data. | DBA / Engineering | Restricted |
| **AS-03** | **User Credentials** | Hashed passwords and associated user data in the `users` table. | AuthService | Restricted |
| **AS-04** | **JWT Secret** | The secret key used to sign all JSON Web Tokens. Stored as an environment variable. | System Administrator | Secret |
| **AS-05** | **Audit Logs** | Logs generated by the `SecurityLoggerInterceptor`, capturing security events. | Security Team | Confidential |
| **AS-06** | **Backup Scripts** | `backup-db.sh` and `restore-db.sh` scripts. | Engineering Team | Internal |
| **AS-07** | **Database Backups** | SQL dump files created by the backup script. | System Administrator | Restricted |
| **AS-08** | **API Documentation** | `API_REFERENCE.md` describing all endpoints. | Engineering Team | Internal |
| **AS-09** | **Cloud Credentials (Optional)** | AWS IAM credentials for S3 and Step Functions access. Stored as environment variables. | Cloud Administrator | Secret |
| **AS-10** | **Deployment Environment** | The runtime environment (e.g., Docker container, EC2 instance) where the backend runs. | DevOps / SRE | Internal |
